# Trend Cards on the Client Command Center

Who this is for: **EHS Managers** and **Company Admins** reading the Command Center dashboard.

## What it is

The Command Center now shows seven trend cards, grouped into three sections, right
below the KPI row:

**Safety Events**
- **Injuries** — count of recorded injuries (first aid, medical treatment, lost-time,
  fatality) over the last 6 months, with a small trend line.
- **CAPAs** — currently open corrective/preventive actions.
- **Audit Findings** — currently open findings from internal/external audits.

**Compliance Health**
- **Recurring Problems** — incident types or audit-finding categories that have shown
  up more than once, so you can spot a pattern instead of treating each event in
  isolation.
- **Upcoming Compliance Gaps** — shows only the single nearest deadline across CAPAs,
  audit findings, chemical SDS/expiration dates, and training certifications (e.g.
  *"Next due: Cal/OSHA annual report — 14 days"*), not a full table. The count on the
  card itself is how many deadlines fall in the next 90 days.

**Chemical & Waste**
- **Chemical Hazard Snapshot** — a qualitative risk level (Negligible → Extreme) based
  on how many chemicals in inventory are scheduled substances or carry
  carcinogen/acute-toxicity hazard statements.
- **Waste Activity** — waste streams logged in the last 6 months.

## Demo data

If an account has zero records across all six underlying data sources (incidents,
CAPAs, audit findings, chemicals, waste streams, training records), the dashboard shows
clearly-labeled sample figures instead of a wall of zeros. Every affected card gets a
visible banner at its top — an icon plus the text **"Sample data — your live data will
appear here"** — never just a footnote or a color change. As soon as any real record
exists in any of the six sources, the banner disappears and the cards switch to real
numbers.

## Empty states

Individual cards never show a bare `0`, a dash, or "N/A" when data is genuinely absent.
Each card has its own reassuring, specific empty message, for example:

- *"No injuries recorded — safety record looks good."*
- *"No open CAPAs — all corrective actions closed."*
- *"No chemical risk flags found — inventory looks good."*

## Card insights

Each card includes a one-line "so what" summary underneath the number (for example,
*"2 recurring issues detected — review for a common root cause"*). These summaries are
generated by a single batched AI call across all 7 cards at once. If the AI provider is
unavailable, unconfigured, or the request fails, every card instead gets a deterministic
sentence built directly from its own numbers — the dashboard never shows a blank or
broken summary.

## How the nearest compliance deadline is chosen

The dashboard gathers every future-dated compliance item it can find — CAPA due dates,
audit-finding due dates, chemical SDS-review/expiration dates, training certification
expirations — and picks the single one with the fewest days remaining. Anything already
overdue is excluded (it should show up in the relevant module's own overdue view, not
here), and anything more than 90 days out doesn't count toward the card's "how many
gaps" number, though it's still eligible to become "the nearest one" once closer items
close out.

## Notes for verification before a client demo

- This is a read-only aggregation of existing tables through the same tenant-scoped
  repository functions every other dashboard panel uses — no database migration, no new
  tables, no change to RLS.
- Spot-check the AI-written summaries in staging before showing a live account to a
  client — they're generated from real numbers, but should still read naturally and
  avoid overstating a regulatory claim.
- Confirm the DEMO_TRENDS sample values (in
  `src/lib/actions/getDashboardTrends.ts`) still read as plausible-but-obviously-not-live
  before using an empty account in a client-facing demo.
